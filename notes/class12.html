Browser Events

Browser Events

• Events are actions users perform in the browser:

⇒ Click

⇒ Key press / key down / key down

⇒ Form submit

⇒ Scroll

• We can use JS to listen and respond to events

Browser Events

1) Use document.querySelector() to select the element you want to apply the event to:


  document.querySelector('#click-me')

2) Apply the event (ex: .onclick) to the query:


  document.querySelector('#click-me').onclick
Browser Events

3) Set the query/event equal to a function


  document.querySelector('#click-me').onclick = hello;
  
  function hello() {
    alert('Hello, World!');
  }

• Function hello runs when #click-me is clicked

innerHTML Property

innerHTML Property

• .innerHTML is used to read and write HTML

• Assuming the following HTML:


  <div id="result">Hello, World!</div>

⇒ To read:


  document.querySelector('#result').innerHTML;
  => 'Hello, World!'

⇒ To write:


  document.querySelector('#result').innerHTML = 'Cats';

Input Elements

Input Elements

• Input elements enable interactive behaviors

• Allow users to input, select, check, and submit data

• Two (primary) input-related elements:

<input>

<select>

• Also see: <textarea>

<input> element

• Used to accept a variety of data from user


  <input type="text" />

• The type attribute determines the type of input:

⇒ text (default; don't have to specify)

⇒ password

⇒ radio

⇒ checkbox

⇒ color

⇒ date
